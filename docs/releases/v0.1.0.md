# TinyML_NPU v0.1.0

TinyML_NPU v0.1.0 是首个面向研究和教学的公开版本。它提供一条可检查的 INT8 TinyML NPU 链路：受约束的 ONNX QDQ compiler、32-byte VenusCore uOP、SpinalHDL RTL、Zynq bare-metal runtime，以及在原版 Digilent Zybo XC7Z010 上验证的 KWS testvector 闭环。

## Release Highlights

- 1 cluster、4 lanes、SIMD4 的 VenusCore INT8/INT32 数据通路。
- APB3 control、AHB-Lite DMA 和 128 KiB shared BRAM integration。
- 44-uOP KWS bundle，PS runtime relocation 和结构化 JTAG result ABI。
- Python compiler tests、behavioral simulator、glue RTL protocol regressions 和本地 `make check`。
- Vivado/Vitis 2021.1 非交互构建与实体板自动验收。
- 面向学习者的架构、compiler、ISA、runtime、bring-up 和验证文档。

## Assets

- `tinyml_npu_zybo7010_v0.1.0.bit`：可直接下载到 PL 的 bitstream。
- `tinyml_npu_zybo7010_v0.1.0.xsa`：包含 bitstream 的 Vivado hardware platform。
- `tinyml_npu_kws_test_v0.1.0.elf`：bare-metal KWS testvector application。
- `MANIFEST.json`、`SHA256SUMS` 和 `VERIFICATION.txt`：来源、校验和与验收摘要。

这些二进制由 AMD Xilinx Vivado/Vitis 2021.1 生成，可能包含受 vendor terms 约束的生成内容。Apache-2.0 适用于仓库原创源码，不覆盖第三方和 vendor 权利。下载或再分发前请阅读 `THIRD_PARTY_NOTICES.md`。

## Known Boundaries

- 仅支持原版 Digilent Zybo XC7Z010，不支持 Zybo Z7。
- 仓库不分发模型、训练代码或数据集。
- 演示只使用固定的 120-sample KWS testvector，没有实时音频输入。
- Compiler 是受约束的研究实现，不是通用 ONNX runtime。
- 仓库不启用 GitHub-hosted CI；本 Release 的 manifest 记录本地 `make check`、Vivado/Vitis 和实体板验收。

## English Summary

v0.1.0 is the first research and teaching release of TinyML_NPU. It publishes an inspectable compiler-to-FPGA path and a physically validated 120-sample KWS testvector demo on the original Digilent Zybo XC7Z010. The release passes 31 compiler tests, protocol-level glue RTL regressions, 50 MHz implementation timing, and the structured physical-board acceptance test with 120/120 reference top1 matches and 117/120 label accuracy. Generated vendor artifacts carry separate notices.
