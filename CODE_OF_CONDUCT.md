# Code of Conduct

TinyML_NPU contributors are expected to keep technical discussion respectful, specific, and focused on improving the project.

## Expected Behavior

- Review ideas and code on their technical merits.
- Give reproducible evidence when reporting failures or performance changes.
- Be clear about uncertainty and distinguish measured results from assumptions.
- Respect licensing, authorship, privacy, and responsible disclosure requirements.
- Accept correction and keep disagreement professional.

## Unacceptable Behavior

Harassment, discriminatory language, personal attacks, deliberate disruption, publication of private information, or knowingly misleading technical claims are not accepted in project spaces.

## Enforcement

Report conduct concerns privately through the repository maintainer's GitHub profile. Maintainers may edit or remove content, close discussions, or restrict participation when necessary. Decisions should be proportional, documented privately where appropriate, and applied consistently.
